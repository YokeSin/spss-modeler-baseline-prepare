# セクション別 Markdown スニペット例

[phase2-summary-report.md](../phase2-summary-report.md) で生成する `<RUN_DIR>/reports/summary_report.md` の各章に貼る見本。
**このファイルの中身を summary_report.md に丸ごとコピーしないこと。** 必要なスニペットだけ抜き出して、`outputs/` 配下の HTML から読み取った実値で置き換えてから貼る。

数値・文字列はすべて出力 HTML から読み取った実値で書く。読めなかった値は `-` または `(取得失敗)` と明示する。

---

## ヘッダー (タイトル + 前提メタデータ)

```markdown
# SPSS Modeler 分析サマリー — 患者データ薬品処方予測

- 生成日: 2026-05-10
- 入力データ: `C:/data/drugs.csv` (200 レコード × 7 列)
- タスク種別: 分類 (5 クラス)
- ターゲット: 薬品
- 想定意思決定: 処方薬品の事前推奨 (確認済み)
```

タスク別の書き分け:
- 分類: `タスク種別: 分類 (N クラス)` / `ターゲット: <列名>`
- 数値予測: `タスク種別: 数値予測` / `ターゲット: <列名>`
- クラスタリング: `タスク種別: クラスタリング (K クラスタ)` / `ターゲット: なし`

---

## 1. 分析の目的と前提

```markdown
## 1. 分析の目的と前提

- **分析目的**: 患者属性から処方薬品 (5 種) を予測し、初診時の処方推奨に活用する。
- **想定意思決定**: 医師の処方判断を補助する候補提示 (最終判断は医師)。
- **タスク種別**: 分類 (5 クラス)
- **ターゲット変数**: 薬品 (採用根拠: ユーザーが明示)
- **置いた仮定**:
  - CSV 区切り文字: カンマ / ヘッダ: あり / 文字コード: SystemDefault
  - Partition: 学習 70% / テスト 30% (random_seed=12345)
- **対象モデル (5 件)**: ロジスティック回帰 / ニューラル ネット / Tree-AS / XGBoost ツリー / QUEST
```

意思決定が未確認の場合:

```markdown
- **想定意思決定**: 未確認 (本レポートでは精度水準のみ評価する)
```

## 3. 分析結果から読み取れること

### 3.1 データの特徴 (dataaudit より)

```markdown
### 3.1 データの特徴

- レコード件数: 200 件
- フィールド数: 7 (連続型 4 / フラグ型 1 / 名義型 2)
- 欠損: なし (全フィールド 100.000% 有効)
- 注目フィールド:
  - **年齢**: 15〜74 歳, 平均 44.3 歳 (歪度 0.030, ほぼ正規)
  - **薬品**: 5 クラス分布 (Y が約 45%, X が約 27%, A/B/C が少数派)
  - **ナトリウム値 / カリウム値**: 連続変数, 0.0〜0.9 程度のスケール
- 個人情報候補: 無し (氏名・住所等は含まれない)
```

### 3.2 モデル比較 (analysis より、分類・数値予測のみ)

★BEST はテスト指標最良かつ学習-テスト乖離 ≤2 pt のものに 1 つだけ。決め手がなければ無印。

```markdown
### 3.2 モデル比較

| モデル | 学習 | テスト | 乖離 | 備考 |
|---|---|---|---|---|
| ロジスティック回帰 ★BEST | 100.00% | 97.22% | 2.78 pt | テスト最良。乖離はやや大きいが許容範囲 |
| ニューラル ネット | 96.09% | 94.44% | 1.65 pt | 汎化性能良好 |
| Tree-AS | 99.22% | 91.67% | 7.55 pt | 過学習傾向 |
| XGBoost ツリー | 100.00% | 91.67% | 8.33 pt | 過学習傾向 |
| QUEST | 95.31% | 88.89% | 6.42 pt | やや精度低 |
```

数値予測の場合の例:

```markdown
| モデル | 学習 R² | テスト R² | 乖離 | 備考 |
|---|---|---|---|---|
| <モデル名> ★BEST | 0.812 | 0.798 | 0.014 | ... |
```

### 3.3 混同行列 / 残差傾向 (分類・数値予測のみ)

詳細表 (全モデルの一致行列) は `analysis_baseline.html` を参照する形にして、Markdown には所見だけ書く。

```markdown
### 3.3 ベストモデルの誤分類傾向

詳細は [analysis_baseline.html](../outputs/analysis_baseline.html) を参照。

ロジスティック回帰のテスト誤分類:
- 薬品B → 薬品A (1件)
- 薬品B → 薬品Y (1件)

少数クラス (薬品B) で誤分類が集中している。クラス偏りが影響している可能性がある。
```

数値予測の場合は「残差傾向」に置き換える:

```markdown
### 3.3 残差傾向

詳細は [analysis_baseline.html](../outputs/analysis_baseline.html) を参照。
予測値 -- 実測値の誤差は概ね対称分布。X 値が大きい領域で残差が大きくなる傾向あり。
```

### 3.4 評価グラフ所見 (分類のみ)

評価グラフは画像として埋め込む。読みは Gains チャートの一般論 + 過学習リスク。

```markdown
### 3.4 評価グラフ所見

![評価グラフ](../outputs/_tmp_evaluation_baseline.jpg)

上位 20% で全正解の約 90% を捕捉できており、ターゲットの絞り込み効果は十分。
ただし、テスト精度 97.22% は学習データへの依存度が比較的高い ($L で 100% 正解) ため、
未知データに対しては 5〜10 ポイント程度の精度低下を想定して運用する。
```

クラスタリングの場合は 3.2〜3.4 を「クラスタ概況」1 章に置き換え:

```markdown
### 3.2 クラスタ概況

- クラスタ数: K = 5 (AutoCluster 自動決定)
- 主要分離変数: 年齢, ナトリウム値
- 各クラスタのサイズ: C1: 60件 / C2: 45件 / ...
- BEST モデル: クラスタリングのため判定なし
```

## 4. ビジネスへの示唆

3 段落構成。各段落 2〜4 文。重要キーワードは `**bold**` で強調する。

```markdown
## 4. ビジネスへの示唆

**何が分かったか** — ロジスティック回帰がテスト正解率 **97.22%** で最良の性能を示した。
線形モデルが好成績ということは、ターゲット (薬品) と入力変数の関係が概ね線形分離可能であることを示唆している。
ニューラル ネットも 94.44% と僅差で、複雑な相互作用は限定的と考えられる。

**想定意思決定への適合度** — 想定意思決定 (医師の処方判断補助) に対しては、テスト 97.22% は十分な精度水準と評価できる。
ただし、誤分類が少数クラス (薬品B) に集中しているため、本番運用前に少数クラスの recall を別途検証する必要がある。
医師の最終判断を補助する位置づけであれば、現状でも MVP として運用可能。

**データ品質と運用上の留意点** — 母集団 200 件は限定的で、再現性確認のため追加データでの検証が望ましい。
欠損や個人情報の混入は無く、データ品質自体は良好。
クラス偏り (薬品Y が約 45%) があるため、少数クラスの予測根拠を ID 系変数に依存していないかを別途確認したい。
```

書き方ガイド:
- 1段落目「何が分かったか」: ベストモデル + 主要指標 + 業務ドメイン非依存の事実観察
- 2段落目「想定意思決定への適合度」: 意思決定が確認済みならその意思決定への精度の十分さ。**未確認なら「未確認のため精度水準のみ評価」と明記**
- 3段落目「データ品質と運用上の留意点」: クラス偏り / ID 系変数依存 / サンプルサイズ / 欠損 / 個人情報

---

## 5. 次に取るべきアクション

優先度高い順に 3〜5 個。各項目 1〜2 文。

```markdown
## 5. 次に取るべきアクション

1. **少数クラス (薬品B) の予測精度を確認する** — recall を個別に算出し、業務要件 (誤分類許容度) と照合する。
2. **追加データで再現性を検証する** — 別期間 / 別施設データで同程度のテスト精度が出るか確認する。
3. **特徴量重要度を確認する** — ロジスティック回帰の係数を見て、医師が直感的に納得できる説明変数か検証する。
4. **(任意) 過学習傾向のあるモデル (Tree-AS / XGBoost) でハイパラ調整を試す** — テスト精度を改善できる余地があれば再評価する。
```

## 6. 確認ポイント

severity 3 段階で 4〜6 個。チェックリスト形式 (`- [ ]`)。

```markdown
## 6. 確認ポイント

- [ ] **(高)** サンプルサイズの妥当性 — 200 件は限定的。本番投入前に母集団の代表性を確認する。
- [ ] **(高)** ターゲット定義の妥当性 — 薬品列が処方履歴か推奨処方か、業務側と認識合わせをする。
- [ ] **(中)** クラス偏りへの対処 — 薬品Y が約 45% を占める。少数クラスの recall を個別に確認する。
- [ ] **(中)** 過学習傾向の検証 — Tree-AS / XGBoost は学習-テスト乖離 7〜8 pt あり。本番運用には不向き。
- [ ] **(低)** PMML モデルの配備手順確認 — 5 モデルの PMML XML が出力済み。本番環境への配備手順を確認する。
```

severity 選択基準:
- **高**: 本番投入を妨げる可能性がある (サンプルサイズ不足 / ターゲット定義の妥当性 / 個人情報混入)
- **中**: 業務適用前に検証が必要 (クラス偏り / 過学習 / 特徴量妥当性)
- **低**: 運用上の留意点 (配備手順 / 監視体制 / ドキュメント整備)
